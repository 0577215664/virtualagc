Source Code

These source-code files were transcribed from scans made from Don Eyles's personal copy of BURST120 (SUNBURST 120). They were scanned at archive.org's Boston facility, and the scanning was sponsored by Mike Stewart. The code was transcribed from these scans by a team of volunteers who are referenced in the program comments. Comments from the original source code are in ALL-CAPS, whereas comments added later in transcription are in Mixed-Case. In some cases, where similar code blocks exist in previously-transcribed AGC programs (primarily Luminary 99, from Apollo 11) those code blocks were used as a starting point and then corrected to agree with the BURST120 scans. The full scans are available at the Virtual AGC project's collection at archive.org, while more-convenient reduced-size (but reduced-quality) images are available at the main Virtual AGC website. Report any errors noted by creating an issue report at the Virtual AGC project's GitHub repository. Notations on the program listing read, in part:

	YUL SYSTEM FOR AGC: REVISION 0 OF PROGRAM BURST120 
	BY NASA 2021106-031 DEC 7, 1967

	THIS LISTING IS A COPY OF A VERSION OF THE PROGRAM INTENDED 
	FOR USE IN THE ON-BOARD PRIMARY GUIDANCE COMPUTER IN THE 
	UNMANNED FLIGHT OF APOLLO LUNAR MODULE 1 --- THE AS206 MISSION.
Note that the date is the date of the printout, not the date of the program revision.

023669,000015:                                                                                                  #  PROGRAM DESCRIPTION:
023670,000016: 
023671,000017:                                                                                                  #           AUTHOR:  J S MILLER    (MIT INSTRUMENTATION LAB))
023672,000018: 
023673,000019:                                                                                                  #           THIS ROUTINE IS ATTACHED TO T4RUPT, AND IS ENTERED EVERY 960 MS.  ITS FUNCTION IS TO EXAMINE THE STATE
023674,000020:                                                                                                  #  OF THE LOW 8 BITS OF CHAN 32 TO SEE IF RCS-JET FAILURE BITS HAVE APPEARED OR DISAPPEARED.  WHEN A STATE CHANGE
023675,000021:                                                                                                  #  IS DETECTED WHICH PERSISTS FOR 2 SAMPLES (TO FILTER OUT NOISY SIGNALS), AN LMP COMMAND IS SENT (BUT AT MOST ONE
023676,000022:                                                                                                  #  SUCH COMMAND PER SAMPLE) TO ISOLATE THE APPROPRIATE JET PAIR IF THE FAILURE SIGNAL HAS JUST APPEARED.  IF, ON
023677,000023:                                                                                                  #  THE OTHER HAND, THE DISAPPEARANCE OF A FAIL-BIT IS DETECTED, THE EVENT IS PRESUMED TO BE THE RESULT OF THE
023678,000024:                                                                                                  #  GROUND HAVING SENT AN LMP COMMAND TO OPEN THE ISOLATION VALVE AND NO SUBSEQUENT FAILURE HAVING BEEN DETECTED BY
023679,000025:                                                                                                  #  THE DETECTION CIRCUITRY.  IN EITHER CASE, THIS FAILURE-MONITOR PROGRAM RESPONDS BY UPDATING ITS RECORD OF THE
023680,000026:                                                                                                  #  STATE OF THE ISOLATION VALVES, AND ALSO THE PAIR OF WORDS IT MAINTAINS FOR USE BY THE DAP IN THE JET-POLICY
023681,000027:                                                                                                  #  SELECTION LOGIC.
023682,000028: 
023683,000029:                                                                                                  #           A FEW WORDS ABOUT LMP-RESET COMMANDS:  IN THE CYCLE IN WHICH A VALVE-CLOSURE COMMAND IS ISSUED, THE
023684,000030:                                                                                                  #  CORRESPONDING RESET COMMAND IS STORED IN THE REGISTER "JETRESET".  THIS REGISTER IS EXAMINED AT THE BEGINNING OF
023685,000031:                                                                                                  #  EACH RCSMONIT CYCLE, AND IF A COMMAND IS WAITING, IT IS SENT IMMEDIATELY.  THIS CAUSES RESET COMMANDS TO BE
023686,000032:                                                                                                  #  ISSUED AT ABOUT 960 MS AFTER THE SET COMMANDS.  THIS TECHNIQUE ASSURES THAT NO MORE THAN ONE UN-RESET SET-
023687,000033:                                                                                                  #  COMMAND IS ON AT ANY TIME (ALTHOUGH GROUND ACTIONS WILL BE SUPERIMPOSED ON THIS, HOWEVER).
023688,000034: 
023689,000035:                                                                                                  #  CALLING SEQUENCE:
023690,000036: 
023691,000037:                                                                                                  #           TC     RCSMONIT        (IN INTERRUPT MODE, EVERY 960 MS.)
023692,000038:                                                                                                  #  EXIT:
023693,000039: 
023694,000040:                                                                                                  #           TCF    RCSMONEX    (ALL PATHS EXIT VIA SUCH AN INSTRUCTION.  RCSMONEX IS PRESENTLY EQUATED TO RESUME.)
023695,000041: 
023696,000042:                                                                                                  #  ERASABLE INITIALIZATION REQUIRED:
023697,000043: 
023698,000044:                                                                                                  #           VIA SLAP1:  JETRESET          = -0  (RESET-CMD BUFFER EMPTY)
023699,000045:                                                                                                  #                       THISCH32          = +0  (NO FAILURES SENSED YET)
023700,000046:                                                                                                  #                       CH5MASK, CH6MASK  = +0  (0'S IF JETS ARE OK)
023701,000047:                                                                                                  #                       LMPJFAIL          = +0  (ISOLATION VALVES ALL OPEN)
023702,000048: 
023703,000049:                                                                                                  #  OUTPUT:
023704,000050: 
023705,000051:                                                                                                  #           CH5MASK & CH6MASK UPDATED  (1'S WHERE JETS NOT TO BE USED)
023706,000052:                                                                                                  #           THISCH32 SET TO INVERTED LOW 8 BITS OF CHAN 32 (HIGH 7 = 0)
023707,000053:                                                                                                  #           LMPJFAIL UPDATED  (1'S WHEN ISOLATION COMMANDS HAVE BEEN SENT)
023708,000054:                                                                                                  #           JETRESET CONTAINS RESET-COMMANDS TO BE SENT NEXT CYCLE THRU.
023709,000055: 
023710,000056:                                                                                                  #  DEBRIS:
023711,000057: 
023712,000058:                                                                                                  #           A, L, Q, & RCSMONT1 - 4  (PRESENTLY EQUATED TO RUPTREG1 - 4)
023713,000059: 
023714,000060:                                                                                                  #  SUBROUTINES CALLED:
