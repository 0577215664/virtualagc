Source Code

These source-code files were obtained by digitally photographing an Artemis 72 (Apollo 15-17 Command Module) program listing from a private collection and then modifying pre-existing Comanche 55 (Apollo 11) source files to incorporate changes. Photography was by Ron Burkey, and transcription was performed by a team of volunteers. Note that the page images presented online are of reduced quality, and that additional images may be available. Report any conversion errors or legibility problems in page images to info@sandroid.org. Notations on the program listing read, in part:

	GAP: ASSEMBLE REVISION 072 OF AGC PROGRAM ARTEMIS 
	BY NASA 2021114-011 11:40 FEB. 26, 1971
Note that the date is the date of the printout, not the date of the program revision.

010578,000022:                                                                                                  #  PROGRAM NAME     -     SXTMARK
010579,000023: 
010580,000024:                                                                                                  #  MOD. 1        8 AUG. 69       BY P.RYE
010581,000025: 
010582,000026:                                                                                                  #  FUNCTIONAL DESCRIPTION
010583,000027: 
010584,000028:                                                                                                  #        SXTMARK IS CALLED BY P03 AND BY P22, P23, P51, AND P52 VIA R53.
010585,000029:                                                                                                  #        THE REGISTER MARKINDX IS SET TO 5 IF THE CALLING PROGRAM IS P22, OR TO 1 FOR OTHER USERS.
010586,000030:                                                                                                  #           THIS REGISTER INDICATES THE NUMBER OF MARKS DESIRED.
010587,000031:                                                                                                  #        THE REGISTER EXTVBACT IS CHECKED (SUBROUTINE TESTMARK) TO INSURE THAT THE MARK DISPLAY SYSTEM
010588,000032:                                                                                                  #           IS FREE.  IF NOT, BAILOUT (31211) IS DONE.
010589,000033:                                                                                                  #        BIT 2 OF THE REGISTER EXTVBACT IS SET BY SXTMARK, RESERVING THE MARKING AND EXTENDED VERB SYSTEM.
010590,000034:                                                                                                  #        SXTMARK DISPLAYS A FLASHING VERB 51, CALLING FOR MARKS.  A PROCEED RESPONSE TO THIS DISPLAY
010591,000035:                                                                                                  #           WILL RELEASE THE MARKING SYSTEM (SUBROUTINES MKRELEAS, CLEARMARK) AND RETURN TO THE
010592,000036:                                                                                                  #           CALLING PROGRAM.
010593,000037: 
010594,000038:                                                                                                  #        WHEN SUFFICIENT MARKS HAVE BEEN MADE (MARKINDX = 0) A FLASHING V50N25, R1 = 16, IS DISPLAYED.
010595,000039:                                                                                                  #           A PROCEED RESPONSE TO THIS DISPLAY WILL RETURN TO THE CALLING PROGRAM AFTER RELEASING THE
010596,000040:                                                                                                  #           MARKING SYSTEM.
010597,000041: 
010598,000042:                                                                                                  #        MARKS ARE PROCESSED BY THE ROUTINE MARKRUPT AS FOLLOWS:
010599,000043:                                                                                                  #                IF NO MARKS ARE CALLED FOR, ALARM CODE 114 IS SENT AND THE MARKRUPT ROUTINE EXITS.
010600,000044:                                                                                                  #                IF A MARK IS ACCEPTED, MARKFLG IS SET TO ENABLE A REJECT.
010601,000045:                                                                                                  #                IF R21(P20) IS RUNNING, DATA IS MOVED FROM STORAGE MRKBUF1 INTO MRKBUF2.  NEW MARK DATA
010602,000046:                                                                                                  #                        IS STORED INTO MRKBUF1.
010603,000047:                                                                                                  #                IF P22 IS RUNNING, MARK DATA IS STORED INTO SVMRKDAT, USING THE REGISTER P22DEX AS AN
010604,000048:                                                                                                  #                        INDEX AND THE REGISTER 8NN AS A COUNTER.  MARKINDX IS DECREMENTED.
010605,000049:                                                                                                  #                IF R57 IS RUNNING, MARK DATA IS STORED INTO MARKDOWN FOR DOWNLINK, THEN PROCESSED AS FOR
010606,000050:                                                                                                  #                        R21.  MARKRUPT THEN CALLS MARKDISP (IN R57).
010607,000051:                                                                                                  #                FOR OTHER USERS, MARK DATA IS STORED INTO MRKBUF1 AND MARKINDX IS DECREMENTED.
010608,000052: 
010609,000053:                                                                                                  #        MARK REJECTS ARE PROCESSED BY THE ROUTINE MARKRUPT AS FOLLOWS:
010610,000054:                                                                                                  #                IF MARKFLG IS CLEAR (I.E., NO MARK WAS TAKEN), ALARM CODE 110 IS SENT AND THE ROUTINE EXITS.
010611,000055:                                                                                                  #                OTHERWISE, MARKINDX IS INCREMENTED, THE P22 INDICATORS 8NN AND P22DEX ARE DECREMENTED
010612,000056:                                                                                                  #                        (IF THE USER IS P22), AND THE V51FL DISPLAY IN SXTMARK IS REESTABLISHED.
010613,000057: 
010614,000058:                                                                                                  #        IF THE ERASABLE REGISTER CDUCHKWD IS SET TO A NON-ZERO VALUE, VALIDITY OF THE MARKS IS CHECKED
010615,000059:                                                                                                  #           BY THE MARKRUPT ROUTINE AS FOLLOWS:
010616,000060:                                                                                                  #                IF THE CDU'S CHANGE BY MORE THAN 3 BITS OVER THE TIME PERIOD INDICATED BY THE VALUE OF
010617,000061:                                                                                                  #                        CDUCHKWD, ALARM CODE 121 IS SENT AND THE ROUTINE EXITS.
010618,000062:

010620,000064: 
010621,000065:                                                                                                  #        TC      BANKCALL
010622,000066:                                                                                                  #        CADR    SXTMARK
