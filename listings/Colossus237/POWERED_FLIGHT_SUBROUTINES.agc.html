056203,000021:                                                                                                 #      CDUTRIG, CDUTRIG1, CDUTRIG2, AND CD*TR*GS ALL COMPUTE THE SINES AND
056204,000022:                                                                                                 #  COSINES OF THREE 2'S COMPLEMENT ANGLES AND PLACE THE RESULT, DOUBLE
056205,000023:                                                                                                 #  PRECISION, IN THE SAME ORDER AS THE INPUTS, AT SINCDU AND COSCDU.   AN

056207,000025:                                                                                                 #  ROUTINES GO OUT OF THEIR WAY TO LEAVE THE MPAC AREA AS THEY FIND IT,
056208,000026:                                                                                                 #  EXCEPT FOR THE GENERALLY UNIMPORTANT MPAC +2.   THEY DIFFER ONLY IN
056209,000027:                                                                                                 #  WHERE THEY GET THE ANGLES, AND IN METHOD OF CALLING.
056210,000028: 
056211,000029:                                                                                                 #      CDUTRIG (AND CDUTRIG1, WHICH CAN BE CALLED IN BASIC) COMPUTE THE
056212,000030:                                                                                                 #  SINES AND COSINES FROM THE CURRENT CONTENTS OF THE CDU REGISTERS.
056213,000031:                                                                                                 #  THE CONTENTS OF CDUTEMP, ETC., ARE NOT TOUCHED SO THAT THEY MAY
056214,000032:                                                                                                 #  CONTINUE TO FORM A CONSISTENT SET WITH THE LATEST PIPA READINGS.
056215,000033:

056217,000035: 
056218,000036:                                                                                                 #      CD*TR*GS FINDS CDU VALUES IN CDUSPOT RATHER THAN IN CDUTEMP.   THIS
056219,000037:                                                                                                 #  ALLOWS USERS TO MAKE TRANSFORMATIONS USING ARBITRARY ANGLES, OR REAL
056220,000038:                                                                                                 #  ANGLES IN AN ORDER OTHER THAN X Y Z.   A CALL TO THIS ROUTINE IS

056222,000040:                                                                                                 #  MODES (SMNB OR NBSM).   SINCE AX*SR*T EXPECTS TO FIND THE SINES AND
056223,000041:                                                                                                 #  COSINES IN THE ORDER Y Z X THE ANGLES MUST HAVE BEEN PLACED IN CDUSPOT
056224,000042:                                                                                                 #  IN THIS ORDER.   CD*TR*GS NEED NOT BE REPEATED WHEN AX*SR*T IS CALLED
056225,000043:                                                                                                 #  MORE THAN ONCE, PROVIDED THE ANGLES HAVE NOT CHANGED.   NOTE THAT SINCE
056226,000044:                                                                                                 #  IT CLOBBERS BUF2 (IN THE SINE AND COSINE ROUTINES) CD*TR*GS CANNOT BE
056227,000045:                                                                                                 #  CALLED USING BANKCALL.   SORRY.
